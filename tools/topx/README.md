# topx

简洁的系统监控 TUI。一屏看清 **CPU / 内存 / 网络 / 磁盘** + 进程列表，单键交互。

不像 htop 那样信息爆炸，只把最常看的四个维度做成 sparkline，下方留给进程。

## 用法

```bash
topx                  # 默认 1s 刷新
topx -i 2             # 2s 刷新一次（CPU 占用更低）
topx -Q 70            # 4 象限占 70% 高度（默认 50）
```

## 单键操作

| 键              | 动作 |
|-----------------|------|
| `q` / `Esc`     | 退出 |
| `Space`         | 暂停 / 恢复刷新 |
| `↑` `↓` / `j` `k` | 上下移动选中进程 |
| `PgUp` / `PgDn` | 按页翻动 |
| `Home` / `End`  | 跳到首 / 末 |
| `K`             | 杀掉当前选中进程（先确认 y/N） |
| `c`             | 按 CPU 排序 |
| `m`             | 按内存（RSS）排序 |
| `p`             | 按 PID 排序 |
| `t`             | 按运行时长排序 |
| `/`             | 过滤模式（输入字符串过滤命令名 / 用户） |
| `+` / `-`       | 增减 4 象限占总高度的比例（5% 一档，20%–80%） |
| `?`             | 弹出帮助 |

## CLI 参数

| 选项                | 说明                                  | 默认 |
|---------------------|---------------------------------------|------|
| `-i, --interval N`  | 系统数据刷新间隔秒数                  | 1.0  |
| `-Q, --quad-pct N`  | 4 象限占总高度的百分比（20–80）       | 50   |

## 颜色规则

| 维度         | 阈值 / 颜色 |
|--------------|--------------|
| CPU 总占用   | ≥80 红，≥50 黄，≥20 绿，<20 灰 |
| 内存使用率   | ≥90 红，≥75 黄，<75 绿 |
| 进程 %CPU    | 同上 CPU 规则 |
| 进程 %MEM    | 同上内存规则 |
| 网络下行     | 绿（↓）|
| 网络上行     | 黄（↑）|
| 磁盘 I/O     | 青 |

## 数据源（macOS 自带，零外部依赖）

| 维度 | 命令 | 说明 |
|------|------|------|
| CPU / 内存 / 负载 | `top -l 1 -n 0`              | 单次快照，~200ms |
| 网络字节计数      | `netstat -ib`                 | 原始累计字节，相邻两次做差算速率 |
| 磁盘 I/O          | `iostat -d -K -w 1 -c 2`      | 直接给 per-second 值，每次调用阻塞 1s |
| 进程列表          | `ps -eo pid,user,pcpu,...`   | 每 ~2s 刷新一次 |

> 注：macOS `iostat` 不区分读 / 写，只给总吞吐。Linux 版本可以分别显示，未来可加。

实现要点：**所有采集都在后台线程**，主线程只负责渲染 + 键盘，按键不会被 IO 阻塞。

## 依赖

- `python3`（macOS 自带，含 stdlib `curses`）
- `top` / `ps` / `netstat` / `iostat`（macOS / Linux 自带）
