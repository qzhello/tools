# top2

简洁的系统监控 TUI。一屏看清 **CPU / 内存 / 网络 / 磁盘** + 进程列表，单键交互。

不像 htop 那样信息爆炸，只把最常看的四个维度做成 sparkline，下方留给进程。

## 用法

```bash
top2                  # 默认 1s 刷新
top2 -i 2             # 2s 刷新一次（CPU 占用更低）
```

## 单键操作

| 键              | 动作 |
|-----------------|------|
| `q` / `Esc`     | 退出 |
| `Space`         | 暂停 / 恢复刷新 |
| `↑` `↓` / `j` `k` | 上下移动选中进程 |
| `PgUp` / `PgDn` | 按页翻动 |
| `Home` / `End`  | 跳到首 / 末 |
| `K`             | 杀掉当前选中进程（先确认 y/N） |
| `c`             | 按 CPU 排序 |
| `m`             | 按内存（RSS）排序 |
| `p`             | 按 PID 排序 |
| `t`             | 按运行时长排序 |
| `/`             | 过滤模式（输入字符串过滤命令名 / 用户） |
| `?`             | 弹出帮助 |

## 颜色规则

| 维度         | 阈值 | 颜色 |
|--------------|------|------|
| CPU 总占用   | ≥80 红，≥50 黄，≥20 绿，<20 灰 |
| 内存使用率   | ≥90 红，≥75 黄，<75 绿 |
| 进程 %CPU    | 同上 CPU 规则 |
| 进程 %MEM    | 同上内存规则 |
| 网络下行     | 绿（↓）|
| 网络上行     | 黄（↑）|
| 磁盘读       | 青（R）|
| 磁盘写       | 紫（W）|

## 实现说明

- 系统数据：`top -l 1 -n 0` 一次解出 CPU / 内存 / 网络 / 磁盘累计字节
- 进程列表：`ps -eo pid,user,pcpu,pmem,rss,etime,command`
- 网络 / 磁盘速率：用相邻两次累计值做差除以时间
- **后台线程采集**，主线程只负责渲染 + 键盘，键响应不会被阻塞
- 终端 < 60×14 时显示提示，请放大窗口

## 依赖

- `python3`（macOS 自带，含 stdlib `curses`）
- `top`、`ps`（macOS / Linux 自带）
